# Architecture — VLSCOA Website

> Generated by `/codebase-overview` on 2026-07-12. Freely editable — update when structure changes.

## Tech Stack

| Layer | Technology |
|-------|------------|
| Static site generator | Eleventy (11ty) v3 (`@11ty/eleventy` ^3.0.0 — the only dependency) |
| Templates | Nunjucks (`.njk`); Markdown rendered through Nunjucks (`markdownTemplateEngine: "njk"`) |
| Styling | Single stylesheet `src/assets/styles.css` — no framework |
| Client JS | None beyond the trivial (deliberate) |
| CMS | Sveltia CMS at `/admin` (`src/admin/`), Git-backed, publishes directly on save |
| Hosting | Cloudflare Pages — builds `npm run build` → `_site` on push to `main`; Cloudflare DNS/CDN/TLS |

~50 source files: 13 `.njk` templates, 14 recorded PDFs, 19 `.md`, 5 `.json`, 1 `.css`.

## Build & Config

- **`.eleventy.js`** — the only build config:
  - `dir`: input `src`, output `_site`, includes `_includes`, data `_data`
  - Passthrough copy: `src/assets`, `src/documents`, `src/admin`, `src/robots.txt`, `src/_headers`
  - Filters: `instrument` (look up an instrument by `id`, e.g. `"02.1"`), `isoDate`, `longDate` (en-US, UTC)
- **`src/_headers`** — Cloudflare Pages security headers (nosniff, SAMEORIGIN, referrer-policy)
- **No `wrangler.toml`** — Pages project, not a Worker; adding one breaks the build
- Scripts: `npm start` (serve, :8080 live reload), `npm run build`
- No env vars, no secrets, no tests — the build itself is the check

## Module Map

| Directory | Purpose | Notes |
|-----------|---------|-------|
| `src/_data/instruments.json` | **Single source of truth** for all 14 recorded instruments | Fields: `id`, `group` (plats/master/coa/manual/guidelines), `title`, `summary`, `docNo`, `opr`, `pdf`, `county`, `reading` (null for plats) |
| `src/_data/site.json` | Association settings: contacts, form URLs, nav | |
| `src/_includes/` | Layout chain: `base.njk` → `page.njk` / `reading.njk` | `reading.njk` carries the "unofficial — recorded instrument controls" banner |
| `src/reading/` | 11 markdown reading copies of instruments | Auto-transcribed; `[sic]` marks OCR garble; §18.3 "Travis County" scrivener's error preserved verbatim |
| `src/documents/` | 14 recorded PDFs | Filenames must match `instruments.json` `pdf` field exactly; see its README.md |
| `src/content/notices/` | Announcements (CMS-managed) | Directory data file `notices.json` |
| `src/content/minutes/` | Approved minutes (CMS-managed) | Directory data file `minutes.json`; approved only, never drafts |
| `src/admin/` | Sveltia CMS (`config.yml` + `index.html`) | `config.yml` must stay in sync with content folders + directory data files |
| `src/assets/` | `styles.css` (design tokens in `:root` custom properties) | Mirrored in `.ai/branding/tokens.json` |

## Key Patterns

- **Data-driven rendering**: templates never hard-code legal facts; everything about an instrument (doc numbers, county cites, PDF links) flows from `instruments.json` via the `instrument` filter. Corrections happen in data, not templates.
- **Two content authorship paths**: repo markdown edits and CMS saves both land as commits on `main` (audit trail is a governance feature); either path triggers a Pages deploy.
- **Layout inheritance**: `base.njk` (chrome + footer disclaimers on every page) → `page.njk` (standard pages) / `reading.njk` (reading copies with the controls-banner + county-record link).
- **Verification**: check the live site (`curl -sI https://www.vlscoa.org/...`) after deploys, especially PDF content-types — not just the local build.
